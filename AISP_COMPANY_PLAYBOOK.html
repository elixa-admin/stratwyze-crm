AISP Overview

Agent — Autonomous Execution

You execute work. Not strategize, not approve, not plan. Execute.

How agent autonomy works:

  • You get a goal with success criteria (objective, measurable)
  • You plan your approach
  • You execute
  • You observe: does it meet criteria?
  • If YES: done, report result
  • If NO: refine approach, loop back to step 2
  • Loop max 5 times; escalate to director if still not done

Remember: You have memory. You learn from feedback. You improve over time.

AISP's Autonomous Systems

Four interlocking systems enable agents to run autonomously, improve over time, and stay reliable.

1. Smart Toolkit (Pattern Library)

What it does: Detects problems you've solved before. Auto-suggests or auto-runs fixes.

  • Stores bug fixes, solutions, learned patterns
  • Confidence scoring (90%+ = auto-run, 85%–89% = suggest, <85% = history only)
  • Shared across all projects (cross-project learning)
  • Syncs weekly to GitHub (persistent, auditable)

Example: "Token expiry" fix had 95% confidence. Next time you encounter "401 Unauthorized", the pattern auto-suggests the fix with 95% confidence. Try it. Works? Remember. Doesn't work? Refine and re-run.

2. Automated Agent Creation

What it does: When you notice a recurring task, the system detects it and suggests creating an agent.

  • Triggers: standup mention ("I reviewed 50 PRs manually"), sprint repeat, time logging (>5h/week on same task)
  • 2-minute questionnaire → agent spec auto-generated → deployed
  • Zero CLI commands needed (system detects and prompts)

Example: If you say "I synthesized feedback all week" every Friday, the system flags it: "Create agent?" You say yes. Ghost (feedback agent) is born and runs every Friday automatically.

3. Agent Loops (Self-Correcting Execution)

What it does: Agents don't guess when they're done. They iterate until objective success criteria are met.

The loop:

  1. Plan: What approach will hit the criteria?
  2. Execute: Do the work
  3. Observe: Collect output
  4. Verify: Does it meet success criteria?
  5. Iterate: If YES → done. If NO → refine and loop back (max 5 times)

Example: "Review all PRs until: zero critical issues found + 85%+ coverage + recommendations ready." Agent loops, reviews, checks. Missed a critical issue? Loop back, deeper review. Coverage still 82%? Loop back, flag authors. Criteria met? Done.

4. Managed Agents (Cloud-Deployed, 24/7)

What they are: Agent loops running on Anthropic's cloud, scheduled, autonomous.

  • Neo: Daily 6 AM — reviews all PRs from past 24h
  • Ghost: Friday 5 PM — synthesizes weekly feedback
  • Tank: 2x daily (6 AM, 6 PM) — infrastructure health check
  • Memory enabled: learns from feedback, improves each run
  • Results auto-delivered (email, Slack, dashboard)

How They Work Together

The flow: Problem occurs → Smart Toolkit suggests fix (or auto-runs) → If routine, Automated Agent Creation flags it → Agent created runs via Managed Agents on schedule → Uses Agent Loops to verify success → Stores learnings in pattern library → Next time, higher confidence

Common Questions

How do I know if my task is done? ▼
Your director or task spec includes success criteria (objective, measurable). You've completed the task when all criteria are met. Use the Agent Loop: verify against criteria each iteration. If criteria aren't clear, ask your director before starting.
What do I do if I'm stuck? ▼
1. Check Pattern Library (solved it before?). 2. Ask your director (that's their job). 3. If it needs cross-team help, Zion (coordinator) handles it.
How do agents improve over time? ▼
Memory + Feedback. Each agent has a memory file (persisted learnings from past runs). When you give feedback ("this approach worked better" or "I preferred the concise version"), the agent stores that. Next task: remembers what worked before.
What's the token budget? ▼
Per sprint: 10K–15K tokens (Sprints 1–9), ~12K (Sprint 10). Total per Wave: ~300–450K tokens. Smart Toolkit and Pattern Library reuse save 25–50% vs. manual approach.
