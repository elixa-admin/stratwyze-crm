Deployment Tiers

Tier determines which environments your change reaches: Dev-only vs. Dev+Prod.

Tier A: Dev-Only

When: Internal changes. Customers won't see this.

Where: Vercel dev only. Does not merge to main.

Examples: Bug fixes, refactoring, tech debt, docs, internal tooling.

Tier B: Dev+Prod

When: Customer-facing changes.

Where: Vercel dev → verification → merge to main → Vercel prod.

Examples: Features, APIs, customer emails, sales materials, security fixes.

Autonomy Levels

Autonomy determines who approves and what platform handles it. Thresholds: $5K / $50K / $500K.

Autonomous (< $5K)

Definition: Self-contained, low-risk, repeatable tasks.

Approval: None. Webhook executes automatically.

Platform: Codex Mini or Claude Haiku

Examples: Routine bug fixes, template updates, data analysis.

Human-in-Loop ($5K–$50K)

Definition: New patterns, customer-facing, moderate risk.

Approval: You (via Dashboard). Review, approve, or reject.

Platform: Claude Sonnet

Examples: Customer emails, sales pitches, small features.

Senior Review ($50K–$500K)

Definition: Strategic decisions affecting multiple projects.

Approval: You (Dashboard). Full strategy memo required.

Platform: Claude Opus

Examples: Architecture, new features, major refactors.

Board-Level (> $500K)

Definition: Existential company decisions.

Approval: You (Dashboard). Formal memo + final approval.

Platform: Claude Opus

Examples: Funding strategy, M&A, major pivots.

Deployment Architecture

GitHub Branching

Per Project:

  • main = Production (Vercel prod project)
  • dev = Staging (Vercel dev project)

Vercel Projects

Project Branch Environment Trigger
[project]-dev dev Staging Sprint 10 + "Request Dev Deploy" button
[project]-prod main Production Merge from dev + "Promote to Prod" button

Sprint Deployment

Sprints 1–9: Develop locally. Auto-commit to dev branch. No deployment.

Sprint 10: Final sprint. Commit to dev → Dashboard "Request Dev Deployment" → GitHub Action → Vercel dev.

Verification: Run tests. You perform manual smoke test on staging URL.

Promotion: Click "Promote to Prod" → Merge dev to main → Vercel prod deploys automatically.

Rollback (Blue-Green)

Each Vercel deployment keeps the previous version live. If prod breaks, revert the commit and re-deploy. Traffic switches back instantly.
