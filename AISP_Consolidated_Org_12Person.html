Scaling Strategy: Specialize Generalists (Depth-First)

Current Model (14 people):
Each team has 1 senior (specialized) + 1-2 juniors (generalists). Juniors are broad because the scope is limited. They learn through breadth.

At Sprint 6 (16-18 people):
Juniors start showing specialization pressure. Example: CASEY (Developer) is handling full-stack, but needs to split into:
CASEY_SENIOR → Backend Lead
CASEY_JR → Frontend Developer
CASEY_JR2 → API/Integration Developer

At Sprint 12 (20+ people):
Teams are more specialized. Example: BLAKE (DevOps) becomes:
BLAKE_SENIOR → DevOps/SRE Lead
BLAKE_JR → Infrastructure Agent
BLAKE_JR2 → Monitoring & Observability Agent

Key Principle: You don't add new directors or restructure. You add specialists under existing seniors as load grows. This keeps the org structure stable while deepening expertise.
