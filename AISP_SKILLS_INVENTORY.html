Core Skills Available in AISP

Smart Skill Auto-Suggestion System

How AISP automatically recommends and invokes skills based on context.

Detection Triggers

Context / Trigger Detects Auto-Suggests Skill Confidence
"New project incoming" CEO standup mentions new project /launch-your-project 95%
"Complex recurring task" Same task logged >3 times in a sprint /launch-your-agent (or auto-create interactive agent) 85%
"Sprint start" Monday standup or /sprint command /goal (to set sprint objectives) 90%
"Major feature design" Feature spec >500 words or architecture discussion /plan (to design implementation) 80%
"Code change detected" PR created on feature branch /code-review (automatic) 100%
"Pre-deployment review" PR targets main branch (Sprint 10) /ultrareview (gate before merge) 95%
"Sprint execution" Sprint plan document exists + goals set /execute-sprint-build (optional automation) 70% (requires explicit authorization)
"Session start" New session begins /memory (auto-load project context) 100%

Auto-Invocation Decision Tree

IS_NEW_PROJECT ├─ YES → /launch-your-project (95% confidence) └─ NO ↓ IS_SPRINT_START ├─ YES → /goal (90% confidence) └─ NO ↓ IS_CODE_CHANGE ├─ YES → /code-review (100% auto-trigger) │ └─ Sprint 10? → /ultrareview (95% gate) └─ NO ↓ IS_MAJOR_FEATURE ├─ YES → /plan (80% suggestion) └─ NO ↓ IS_RECURRING_TASK ├─ YES & >3x/week → /launch-your-agent (85% suggestion) └─ NO ↓ IS_SESSION_START ├─ YES → /memory (100% auto-load) └─ NO: No suggestion

Confidence-Based Triggering

Confidence Action Example
95%+ Auto-invoke (no user approval) /code-review on PR creation, /ultrareview on main PR
80-94% Suggest + wait for approval "Sprint detected. Run /goal to set objectives?"
70-79% Suggest but don't block "Recurring task detected. Consider /launch-your-agent?"
<70% Don't suggest (too uncertain) No suggestion shown

/goal Integration: Wave Planning & Execution

How /goal integrates throughout the Wave lifecycle to set objectives and measure success.

Wave 1: /goal at 3 Levels

Level 1: Wave Goal (30 sprints)
Triggered: Week 1 of Wave | Set company-wide vision for 30 sprints
Command: /goal wave 1
Questions:
✓ What's the ultimate goal? (Deploy autonomous org)
✓ How do you measure success? (95% autonomy, 30% token savings)
✓ What are key milestones? (Build Phase 1.1 → 1.2 → 1.3)
✓ Success criteria? (All agents trained + live in production)

Output: WAVE_1_PLAN.md with objectives, milestones, KPIs
Level 2: Build Phase Goal (10 sprints)
Triggered: Week 2 of each Build Phase | Set technical goals for 10 sprints
Command: /goal build-phase 1.1
Questions:
✓ Build Phase name? (Foundation & v1.0)
✓ What gets delivered? (Org structure, naming, initial agents)
✓ Tech stack? (Node, Next.js, PostgreSQL)
✓ Team composition? (Trinity, Tank, agent Neo)
✓ Success criteria? (Sprint 10: production deployment verified)

Output: BUILD_PHASE_1.1_PLAN.md with deliverables, tech decisions, acceptance criteria
Level 3: Sprint Goal (1-2 weeks)
Triggered: Weekly (optional) | Set detailed objectives for 1-2 week sprint
Command: /goal sprint 1
Questions:
✓ Sprint name? (Sprint 1: Org Setup)
✓ What specific features/tasks? (2-3 list items)
✓ Team capacity? (Trinity 40h, Tank 30h, Neo auto)
✓ Success metrics? (Tests pass, coverage >85%, 0 critical bugs)
✓ Blockers? (Any known risks?)

Output: SPRINT_1_PLAN.md with tasks, estimates, success criteria

Integration with /execute-sprint-build

Workflow:
  1. /goal sprint N → generates SPRINT_N_PLAN.md
  2. Plan file is created with tasks, estimates, success criteria
  3. /execute-sprint-build N /path → reads plan
  4. System parses objectives from goal
  5. System executes all tasks in parallel (within team capacity)
  6. System verifies against success criteria
  7. System reports: completed ✓, failed ✗, at risk ⚠️

Goal Templates (Auto-Generated by /goal)

When you run /goal, the skill generates a plan file with this structure:

Level Output File Sections
Wave WAVE_1_PLAN.md Vision, 3 build phases, success criteria, KPIs, timeline
Build Phase BUILD_PHASE_1.1_PLAN.md Deliverables, tech decisions, 10 sprints breakdown, acceptance criteria
Sprint SPRINT_1_PLAN.md Tasks, owner, estimates, dependencies, success metrics
