## 🔍 **DESIGN SEARCH METHODOLOGY**

### **Phase 1: Define Design Need**
When starting a new UI component or page, ask:
1. What is the primary user goal?
2. What type of component/page is this? (form, dashboard, card, modal, etc.)
3. What's the tone? (professional, playful, minimal, rich, etc.)

### **Phase 2: Search Resources (in order)**
1. **Component-specific:** Check shadcn/ui, Aceternity, Magic UI for the component type
2. **Dashboard/Analytics:** Check Tremor for data visualization patterns
3. **SaaS patterns:** Check SaaS UI Design, SaaS Frame for similar products
4. **Interaction:** Check Page Flows, Mobbin for user interaction patterns
5. **Visual inspiration:** Check Awwwards, Site Inspire for color/typography
6. **Trends:** Check Recent Design for current trends

### **Phase 3: Extract Design Decisions**
From each resource, identify:
- [ ] Color scheme (primary, secondary, accent colors)
- [ ] Typography (font families, sizes, weights)
- [ ] Spacing patterns (padding, margins, gaps)
- [ ] Component sizing (button sizes, card dimensions)
- [ ] Interaction states (hover, focus, active, disabled)
- [ ] Animations/transitions (if applicable)

### **Phase 4: Implement with TailwindCSS**
Use the extracted design decisions to:
- Apply colors from our palette
- Use consistent spacing scale
- Create reusable components
- Document in Design System

### **Phase 5: Verify Quality**
- Does it match Stratwyze brand?
- Is it consistent with existing components?
- Does it meet accessibility standards?
- Is it mobile-responsive?
