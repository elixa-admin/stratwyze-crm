(a) Company Structure: The AISP Organization

Every project under AISP inherits this company structure. It's the same 14-person model for all projects.

The 14-Person Organization

BRANDON (CEO) ├─ ANDERSON (CTO) → Trinity, Tank, Neo ├─ NIOBE (CPO) → Morpheus, Ghost ├─ PERSEPHONE (CSO) → Agent Smith, Cypher ├─ DOZER (CFO) → The Architect └─ ZION (Coordinator) - Cross-team

Why This Structure?

  • Scale without hiring: Same org, infinite projects
  • Clear reporting: Every agent reports to a director
  • Decision authority: Clear escalation paths
  • Proven at 14 people: Optimal size for autonomy vs. coordination

For Your New Project

✅ Answer: Your project will automatically inherit this structure. You don't create a new org — you instantiate this same 14-person model for your project.

When you run `/launch-your-project`, the skill will:

  1. Create the project folder (AISP-ProjectName/)
  2. Symlink to company structure docs
  3. Create project-specific memory/config files
  4. Set up project-level agents (spawned, not shared)

(d) Autonomy Level: What Agents Can Decide

Agents operate within clear guardrails. They decide execution; humans decide strategy.

Autonomy by Role

Who Can Decide Autonomously Must Escalate
Agents How to execute a task (methodology, approach, optimization) What to work on (goals), ambiguous criteria, >5 loop iterations
Directors Domain decisions <$50K, team management, KPIs Strategic shifts >$50K, CEO approval
CEO (You) Vision, strategy, all >$50K decisions Board approval for >$500K

Agent Loop: The Autonomy Pattern

Plan → Execute → Observe → Verify → Iterate (max 5x)

Agent gets a goal with success criteria (objective, measurable). Agent loops until criteria are met. If criteria are still unmet after 5 loops, agent escalates to director.

For Your New Project

✅ Answer: When you initialize your project, you'll specify:
  1. Project autonomy level (95% for dev teams, 80% for new products, etc.)
  2. Escalation triggers (what must be human-approved)
  3. Success criteria format (objective, measurable)
  4. Loop iteration limit (default 5, can adjust per project)

(e) How We Scale: Multi-Project Strategy

AISP scales by replicating the same 14-person org structure across unlimited projects, with shared standards and learning.

Scaling Model

AISP/ (Company Portal) ├─ _company_standards/ (shared, symlinked to all projects) ├─ _shared_resources/ (shared KB, patterns, skills) └─ projects/ ├─ AISP-Portal/ (main product) ├─ AISP-ProjectAlpha/ (project 1) ├─ AISP-ProjectBeta/ (project 2) └─ AISP-ProjectGamma/ (project 3+) Each project: ├─ Own GitHub repo ├─ Own Vercel deployment (prod + staging) ├─ Own Linear project ├─ Own Slack channels ├─ Own context window (isolated) ├─ Access to shared standards (via symlinks) ├─ Access to shared pattern library └─ Same 14-person org structure

Scaling Timeline

Wave Duration Projects Goal
Wave 1 30 sprints (~60 weeks) 1 (AISP Portal) Validate org, prove autonomy, 95% automation
Wave 2 30 sprints 3-5 projects Prove replicability, 98% automation, shared learning

Symlinks (Shared Resources)

✅ How they work:

Your project has:

  • _standards/ → points to AISP/_company_standards/ (read-only)
  • _shared_resources/ → points to AISP/_shared_resources/ (access to KB + skills)

When standards change (e.g., DEVELOPMENT_NOMENCLATURE_STANDARD.md), all projects see the update immediately via symlinks. No duplication, single source of truth.

For Your New Project

✅ Answer: The `/launch-your-project` skill will:
  1. Create this exact folder structure in AISP-ProjectName/
  2. Create symlinks to _standards/ and _shared_resources/
  3. Create .env and config files
  4. Initialize memory/ folder with empty JSON files
  5. Create GitHub workflows (.github/)
  6. Copy vercel.json template

(g) Tool/Skill/Plugin Execution: When & How

Tools and skills are invoked at specific lifecycle moments. This prevents chaos and ensures consistency.

Execution Timeline for a New Project

When Trigger What Gets Invoked Outcome
Project Init User: `/launch-your-project` Project init skill (interactive Q&A) Folder structure, git repo, config files created
Agent Creation Recurring task detected (standup, planning) Automated agent creation (via system monitor) New agent spawned under director, memory initialized
Agent Deployment Complex recurring task → needs cloud agent User: `/launch-your-agent` (managed agents) Agent deployed to Anthropic cloud, scheduled
Code Commit Developer: `git push` GitHub Actions (CI/CD, tests, linting) Tests run, coverage checked, deployment triggered
Pattern Match Agent encounters similar problem Smart Toolkit (pattern library query) Suggested solution auto-invoked if 90%+ confidence
Deployment (Sprint 10) Merge to main branch Vercel deployment, smoke tests Production live, metrics collected
Escalation Agent exceeds autonomy threshold Alert to director, escalation channel Director reviews, makes decision, logs feedback
Wave Completion Sprint 30 ends (end of wave) Wave reflection & documentation Lessons learned, metrics, next wave plan

Smart Toolkit Auto-Invocation

When a problem is encountered:
  1. Agent queries pattern library (SQLite)
  2. Search returns matching patterns with confidence scores
  3. If confidence ≥90%: Auto-run the fix
  4. If confidence 85-89%: Suggest to agent (agent chooses)
  5. If confidence <85%: Store as new pattern for learning

For Your New Project

✅ Answer: Skills & tools are invoked AUTOMATICALLY based on triggers:
  • User only runs: `/launch-your-project` (once) + `/launch-your-agent` (when creating managed agents)
  • Everything else is automatic (GitHub Actions, pattern library, escalations, etc.)
  • No manual tool management needed

(h) Kick-Off Prompt: How to Initialize a New Project

The `/launch-your-project` skill is the single entry point for all new projects. It's an interactive questionnaire (like `/launch-your-agent`).

The `/launch-your-project` Skill

Command: `/launch-your-project`

What it does: Interactive questionnaire that gathers project info, then auto-generates folder structure, git repo, config files, and documentation.

The Questionnaire (What Gets Asked)

Section 1: Project Identity
  • Project name: "What's your project called?" (e.g., "ProjectAlpha")
  • Description: "What problem does it solve?" (one paragraph)
  • Launch goal: "What success looks like at launch?" (measurable)
Section 2: Team & Ownership
  • Project owner (Director): "Who leads this? (Anderson/Niobe/Persephone/Dozer)"
  • Team size: "How many people on your core team? (including agents)"
  • Stakeholders: "Who needs to approve decisions? (escalation path)"
Section 3: Technical Setup
  • Tech stack: "What tech? (Node, Python, Next.js, etc.)"
  • GitHub repo name: "Confirmed as: aisp-projectname"
  • Vercel project: "Production + staging? (auto-created)"
Section 4: AISP Integration
  • Autonomy level: "What % autonomous? (80/90/95%)"
  • Wave: "Which wave? (Wave 1, 2, 3+)"
  • Escalation triggers: "What decisions need human approval?"
  • Success criteria: "How do you measure success? (objective, measurable)"
Section 5: Additional Setup
  • Slack channels: "Create project channels? (auto: #dev, #deployments, #alerts)"
  • Linear project: "Track tasks? (auto-created)"
  • Managed agents: "Any cloud agents? (Neo, Ghost, Tank, custom)"

What Gets Auto-Generated After You Answer

The skill will create:
  1. Folder structure: AISP/projects/AISP-ProjectName/ with all subfolders
  2. Git repo: Initialize git, create .gitignore, link to GitHub (aisp-projectname)
  3. Config files: .env, .env.company (symlink), smart_toolkit_config.json, agent_config.json
  4. Documentation: README.md, WAVE_1_PLAN.md template, project-specific docs
  5. Symlinks: _standards/ and _shared_resources/ pointing to company-level
  6. Memory files: toolkit_memory.json, agent_memory.json (initialized, empty)
  7. GitHub workflows: .github/workflows/ with CI/CD templates (ci.yml, deploy.yml, test.yml)
  8. Vercel setup: vercel.json created, linked to Anthropic/Vercel integration
  9. Slack channels: #aisp-projectname-dev, #aisp-projectname-deployments, etc.
  10. Linear project: AISP-PROJECTNAME project created with default workflows
  11. Initial commit: All files committed with message "Initial commit: project setup from AISP template"
